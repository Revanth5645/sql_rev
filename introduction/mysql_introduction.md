## what is mean by data base language;
In data base language we have DDL,DML,DCL,TCL.
## DDL(data defination language);
In ddl it will be able to deal with database schemas and descriptions, of how the data should reside in the database.
## we have some  types in ddl:-
CREATE: To create a database and its objects like (table, index, views, store procedure, function, and triggers)
ALTER:It will  alters the structure of the existing database
DROP: It will be able to delete objects from the database
TRUNCATE: It will remove all records from a table, including all spaces allocated for the records are removed
COMMENT: The comment will add comments to the data dictionary.
RENAME: It will rename an object
## DML(data manipulation language);
The dml deals with data manipulation and includes most common SQL statements such SELECT, INSERT, UPDATE, DELETE, etc., and it is used to store, modify, retrieve, delete and update data in a database. Data query language(DQL) is the subset of “Data Manipulation Language”. The most common command of DQL is SELECT statement. SELECT statement help on retrieving the data from the table without changing anything in the table.
## The types in dml:-
SELECT:It will retrieve data from a database
INSERT: The data will be inserted into the table
UPDATE:It will updates existing data within a table
DELETE: The data will be abale to delete all records from a database table
MERGE: It will UPSERT operation (insert or update)
## DCL(data control language);
which acts as an access specifier to the database.basically to grant and revoke permissions to users in the database
## Some types in dcl:-
GRANT: grant permissions to the user for running DML(SELECT, INSERT, DELETE,…) commands on the table
REVOKE: revoke permissions to the user for running DML(SELECT, INSERT, DELETE,…) command on the specified table
## TCL (Transactional Control Language);
 It will acts as an manager for all types of transactional data and all transactions. Some of the command of TCL are:-
Roll Back: Used to cancel  or Undo changes made in the database 
Commit: It is used to apply or save changes in the database.
